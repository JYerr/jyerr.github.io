Pre-Approved Offer Automation on Amazon SageMaker

Turning a manual, multi-step offer-generation process into a scheduled, self-checking pipeline — freeing analyst time for strategy instead of babysitting scripts.

Situation

The problem

Generating the pre-approved offer base was a manual, multi-day process — someone had to kick off each data pull, watch it run, sanity-check the output by hand, and only then hand it to marketing. As offer cycles got more frequent, that manual overhead stopped scaling with the team's capacity.

Approach

What I did

  • Moved the offer-generation scripts onto a scheduled Amazon SageMaker pipeline, so compute spun up and ran on a fixed cadence instead of needing someone to trigger each step.
  • Automated the credit-bureau data refresh and feature-generation steps that feed the pre-approval scorecard, keeping the targeting population current without a manual pull each cycle.
  • Built a QC "waterfall" into every run — row counts, population checks, and sanity checks on key fields — so a broken upstream table surfaced immediately instead of silently corrupting the offer set.
  • Added a lightweight monitoring layer that emailed QC output to stakeholders after every run, so anomalies got caught the same day rather than at the next manual review.

Outcome

Result

Freed up roughly 30% of the analyst time previously spent running and checking the offer pipeline by hand, redirecting it toward strategy work instead. Just as important: the monitoring layer meant a broken run got caught and flagged the same day, instead of surfacing days later when someone next happened to look.

Takeaways

What I'd flag for next time

Automation is only half the win — a scheduled pipeline that fails silently is worse than a manual process, because nobody's watching it by default. Building the QC waterfall alongside the pipeline, not bolted on afterward, was what actually made the team comfortable letting go of the manual checks.
